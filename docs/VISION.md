# ThanatosGames Portfolio — Vision & Progress Log

> **至上命題：** このサイトを訪れることは、ThanatosGamesというゲームをプレイすることと同義でなければならない。

---

## ビジョン

### 目指す体験

MOTHER2、トマトアドベンチャー、チンクルのもぎたてルッピーランドのように——

- 画面上に主人公が存在し、プレイヤーが操作する
- マップを実際に歩いて移動することで場所を発見する
- NPCに話しかけると世界観が広がる
- ゲームとポートフォリオの境界がない

訪問者は「情報を見に来た」のではなく「ゲームをプレイしている」感覚になる。

### 参照タイトルとその要素

| タイトル | 参照したい要素 |
|---|---|
| **MOTHER2** | トップビューRPG、現実の街並みとファンタジーの融合、ユーモアのあるNPCの台詞 |
| **トマトアドベンチャー** | カラフルで独特な世界観、ギミック性の高いフィールド、ミニゲーム的体験 |
| **チンクルのもぎたてルッピーランド** | 地図・マップとの連動、探索による発見、収集・達成感 |

## 現状アーキテクチャ（2026-04-04 時点）

### フェーズ0（完了）：静的ポートフォリオ

```
index.html（単一ファイル）
├── ワールドマップ（日本＋フィリピン、SVG描画）
├── ゲームアイコン（マーカー）
│   └── クリック → ゲーム詳細ポップアップ（モーダル）
├── HUD（RPGフォント、上下バー）
└── About・Lore ポップアップ
```

**世界観の基盤:** リアル地図とRPGマップを融合した「THANATOS GAMES WORLD」

---

### フェーズ1（完了）：フィールドページへの遷移

```
index.html → [遷移演出] → game.html?id=xxx
```

**実装済み要素:**
- ゲームアイコンをクリックするとズームイン演出でフィールドページへ遷移
- 各ゲーム専用のフィールド背景（SVG、10種類）
- フィールド上にキャラクターを配置（画像）
- キャラクタークリックでRPGダイアログ（タイピングアニメーション、複数台詞対応）
- ゲームアイコン（上部）クリックでゲーム詳細ポップアップ
- 7拠点ライフベースラベル（NISEKO・SHONAN・TENNOJI・ITOSHIMA・CEBU・PARIS・ST.PETERSBURG）

**ギャップ（ビジョンとの差分）:**
- キャラクターは立ち絵のみ、動かない
- プレイヤーキャラクターが存在しない（操作できない）
- マップを「歩いて」移動する体験がない
- フィールドはSVGの静的背景で、インタラクティブ性が低い

## フェーズロードマップ

### フェーズ2：プレイヤーキャラクターの導入
```
概要: ワールドマップ上で主人公キャラクターを矢印キー/クリックで動かせる
技術: Canvas or CSS transform によるキャラクター移動
効果: 「歩いてゲームに近づく」体験
```

**実装候補:**
- キャラクターはドット絵（16x16〜32x32px）
- WASDまたは矢印キーで移動
- ゲームアイコンに近づくと「！」が出てインタラクション可能になる
- クリック移動（MOTHER2方式）

---

### フェーズ3：フィールドのゲーム化
```
概要: game.htmlのフィールドをインタラクティブなミニ体験にする
技術: CSS animation + JS イベント駆動
効果: そのゲームの世界観を「体験」として伝える
```

**実装候補:**
- 電車ゲーム → ホーム上を電車が通過する演出
- ミスドバトル → 食べ物が飛んでくるミニゲーム的演出
- フィールド内の隠しオブジェクト（調べると台詞が出る）

---

### フェーズ4：真のRPG体験
```
概要: サイト全体を1つのRPGとして設計し直す
技術: Canvas 2D or Phaser.js
効果: ThanatosGamesのポートフォリオがそれ自体ゲームになる
```

**構造:**
- タイトル画面（Press Start）
- オープニングムービー
- ワールドマップでの移動・探索
- 町（ゲームフィールド）に入ると詳細が体験できる
- セーブ・ロード（localStorage）

## 開発原則

1. **「ゲームらしいか？」を常に問う**  
   静的な表示より動的なインタラクション。説明より体験。

2. **段階的に没入度を上げる**  
   急激な大改修より、毎回少しずつ「遊べる要素」を足していく。

3. **世界観の一貫性を保つ**  
   フォント（Press Start 2P）、カラーパレット（dark RPG）、HUDデザインは変えない。

4. **リアルとゲームの融合を忘れない**  
   実在する地名・拠点・体験がゲームの地図に落とし込まれていること。
