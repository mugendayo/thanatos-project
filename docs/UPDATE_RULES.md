# 更新ルール — ThanatosGames ドキュメント運用

---

## 基本原則

1. **事実ベースのみ** — ユーザーが明示的に伝えた情報だけを記録する
2. **推測・補完禁止** — 「おそらく」「〜だと思われる」は書かない
3. **未確定は明示** — 不明な項目は空白にせず `[未確定]` と記載する
4. **追記型** — 既存の情報は上書きせず、変更がある場合は更新ログに残す

---

## ステータス定義

| 表記 | 意味 |
|------|------|
| `確定` | ユーザーが明示的に伝えた情報 |
| `[未確定]` | まだ情報がない、または確認が必要 |
| `[要確認]` | 記録はあるが、最新かどうか不明 |
| `[変更あり]` | 以前の情報から更新された |

---

## 更新フロー

```
1. ユーザーが情報を提供する
   ↓
2. Claude が対象ファイルを確認する
   ↓
3. 「確定 / 未確定」を判定する
   ↓
4. 該当箇所を更新 or 追記する
   ↓
5. 更新ログに日付・内容を記録する
   ↓
6. MASTER.md の作品一覧も必要に応じて更新する
```

---

## ファイル構成

```
/docs/
├── MASTER.md              # スタジオ全体の定義（メインファイル）
├── UPDATE_RULES.md        # このファイル（運用ルール）
└── games/
    ├── _TEMPLATE.md       # 新作品追加時のコピー元
    └── [タイトル名].md    # 各作品の詳細ドキュメント
```

---

## 新作品を追加するとき

1. `/docs/games/_TEMPLATE.md` をコピーして `[タイトル名].md` を作成する
2. `MASTER.md` の「作品群（IP一覧）」テーブルに行を追加する
3. 判明している情報だけを記入し、残りは `[未確定]` のままにする

---

## Claude への指示（参照ルール）

- 会話の冒頭または情報整理の依頼時は、まず `/docs/MASTER.md` を参照する
- 特定の作品について話すときは `/docs/games/[タイトル名].md` を参照する
- ユーザーが「覚えていて」と言ったら、このドキュメントを更新する
- ドキュメントにない情報は「記録がありません」と伝え、補完しない
